= Kernel IPsec subsystem bugs

[role="_abstract"]
A defect in the kernel's IPsec subsystem can cause it to lose sync with the IKE daemon. This can lead to discrepancies between negotiated security associations and actual IPsec policy enforcement, disrupting secure network communication.

To check for kernel-level errors, display the transform (XFRM) statistics:

[subs="+quotes"]
....
# *cat /proc/net/xfrm_stat*
....

If any of the counters in the output, such as `XfrmInError`, show a nonzero value, it indicates a problem with the kernel subsystem. In this case, open a link:https://access.redhat.com/support[support case], and attach the output of the command along with the corresponding IKE logs.
