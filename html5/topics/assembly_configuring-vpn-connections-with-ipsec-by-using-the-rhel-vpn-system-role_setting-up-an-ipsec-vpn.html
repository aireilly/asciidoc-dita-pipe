Configuring IPsec VPN connections by using RHEL system roles

Configure IPsec VPN connections to establish encrypted tunnels over untrusted networks and ensure the integrity of data in transit. By using the RHEL system roles, you can automate the setup for use cases, such as connecting branch offices to headquarters.

Note:

The vpn RHEL system role can only create VPN configurations that use pre-shared keys (PSKs) or certificates to authenticate peers to each other.

  • Configuring an IPsec host-to-host VPN with PSK authentication by using the vpn RHEL system role
  • Configuring an IPsec host-to-host VPN with PSK authentication and separate data and control planes by using the vpn RHEL system role
  • Configuring an IPsec site-to-site VPN with PSK authentication by using the vpn RHEL system role
  • Configuring an IPsec mesh VPN with certificate-based authentication by using the vpn RHEL system role
